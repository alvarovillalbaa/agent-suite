# Reporting

Convert raw operational information into a report that helps a reader decide, not just observe.

## Use this skill for

- weekly team updates
- monthly operating reviews
- board or investor updates
- QBRs and business performance summaries
- KPI dashboards turned into narrative
- functional reports for product, GTM, sales, finance, or ops

## Core reporting principle

A good report explains:

- what happened
- why it happened
- what matters now
- what should happen next

If it stops at "what happened", it is incomplete.

## Default output structure

### 1. Headline summary

- 3-5 bullets only
- biggest win
- biggest miss or risk
- key decision or ask

### 2. KPI view

For each important metric:

- current value
- prior value or baseline
- delta
- one-line driver explanation

### 3. What changed

- launches, deals, incidents, personnel, customer signals, or external shifts

### 4. Risks and blockers

- severity
- owner if known
- expected impact
- mitigation or next step

### 5. Decisions and asks

- what needs approval, support, or attention

### 6. Next period focus

- top priorities for the next week, month, or quarter

## Format selection

Adapt tone and depth to the audience:

- **Executive flash**: short, top-down, decision-oriented
- **Board or investor update**: fewer metrics, more business implications
- **Team operating report**: more execution detail and ownership
- **QBR**: trend-oriented, comparative, and strategic

## Reporting rules

- Always define the time window clearly.
- Use exact deltas where possible, not vague trend language.
- Separate metric description from interpretation.
- Call out data quality issues instead of pretending precision.
- Prefer a small number of decision-relevant metrics over a long vanity list.
- If a metric moves materially, explain the likely cause and what will be done next.

## Narrative rules

- Lead with changes and implications, not activity logs.
- Use plain language; avoid dashboard jargon unless the audience already uses it.
- Highlight exceptions, inflections, and surprises.
- Make the ask explicit if executive attention is required.

## Failure modes to avoid

- report reads like a diary
- no comparison point for metrics
- metrics without interpretation
- long wins section with hidden risks
- board report that is too tactical
- team report that is too abstract to act on

## Quality bar

- The period and audience are obvious.
- The report can be skimmed in under two minutes.
- Metrics, drivers, risks, and asks are all present.
- A reader can tell what requires attention right now.
