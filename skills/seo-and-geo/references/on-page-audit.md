## Scoring Rubric

### Section Weight Distribution

| Audit Section | Weight | Max Score | Rationale |
|--------------|--------|-----------|-----------|
| Title Tag | 15% | 15 | Strongest single ranking signal |
| Meta Description | 5% | 5 | CTR impact, not a direct ranking factor |
| Header Structure | 10% | 10 | Content organization, semantic signals |
| Content Quality | 25% | 25 | Strongest holistic ranking factor |
| Keyword Optimization | 15% | 15 | Relevance signals |
| Internal/External Links | 10% | 10 | Authority flow, context signals |
| Image Optimization | 10% | 10 | Accessibility + image search |
| Page-Level Technical | 10% | 10 | Core Web Vitals, mobile, security |

### Scoring Formula

```
Overall Score = Sum of (section_score × section_weight) × 10
```

Where weights: Title 0.15, Meta 0.05, Headers 0.10, Content 0.25, Keywords 0.15, Links 0.10, Images 0.10, Technical 0.10.

**Worked formula example:**

| Section | Score /10 | Weight | Weighted |
|---------|-----------|--------|----------|
| Title Tag | 8 | 0.15 | 1.20 |
| Meta Description | 6 | 0.05 | 0.30 |
| Header Structure | 9 | 0.10 | 0.90 |
| Content Quality | 7 | 0.25 | 1.75 |
| Keyword Optimization | 8 | 0.15 | 1.20 |
| Internal/External Links | 5 | 0.10 | 0.50 |
| Image Optimization | 6 | 0.10 | 0.60 |
| Page-Level Technical | 7 | 0.10 | 0.70 |
| **Total** | | **1.00** | **7.15** |

**Overall Score** = 7.15 × 10 = **71 / 100**

| Score | Grade | Assessment |
|-------|-------|-----------|
| 90-100 | A+ | Exceptional — minor tweaks only |
| 80-89 | A | Strong — a few optimization opportunities |
| 70-79 | B | Good — several areas need attention |
| 60-69 | C | Average — significant improvements needed |
| 50-59 | D | Below average — major issues present |

## Calibration Examples

### Title Tag

**15/15 (Excellent):** "Keyword Research: 7 Proven Methods to Find Low-Competition Keywords | Brand" — 58 chars, keyword at front, benefit-driven, unique, brand at end

**11/15 (Good):** "The Complete Guide to Keyword Research for Beginners" — 52 chars, keyword present but not at front, no brand

**7/15 (Needs Work):** "Keyword Research" — too short (16 chars), no benefit, no brand, generic

**0/15:** No title tag, or empty title

### Header Structure

**10/10 (Excellent):**
```
H1: How to Do Keyword Research: A Step-by-Step Guide
  H2: What Is Keyword Research and Why Does It Matter?
  H2: 7 Keyword Research Methods That Work
    H3: Method 1: Seed Keyword Brainstorming
    H3: Method 2: Competitor Keyword Analysis
  H2: Free Keyword Research Tools Compared
  H2: Keyword Research FAQ
```

**2/10 (Poor):**
```
H1: Welcome to Our Blog
H1: Keyword Research Guide
H3: Some Tips
H2: More Information
```
Multiple H1s, skipped levels, no keyword strategy.

### Content Quality

**25/25:** 2,400-word guide, covers all subtopics competitors cover, original data from author's experiments, statistics from last 12 months, well-formatted with tables/lists/images, author byline with credentials, first-person "I tested" language.

**8/25:** 700-word article on an informational query, missing several subtopics, no original data, several outdated statistics, wall of text, no author byline.

## Issue Resolution Playbook

### Title Tag Issues

| Issue | Impact | Quick Fix |
|-------|--------|-----------|
| Missing title | Critical | Add: "[Primary Keyword]: [Benefit] \| [Brand]" |
| Too long (>60 chars) | Medium | Shorten: move brand to end, remove filler words |
| Too short (<30 chars) | Medium | Expand: add modifier, benefit, or year |
| Missing keyword | High | Rewrite to include keyword in first half |
| Duplicate title | High | Add page-specific modifier to each page |

### Meta Description Issues

| Issue | Impact | Quick Fix |
|-------|--------|-----------|
| Missing | Medium | Write: "[What page covers]. [Key benefit]. [CTA]." (150-160 chars) |
| Too long (>160 chars) | Low | Trim from end; core message in 150 chars |
| Missing keyword | Low | Incorporate naturally |
| No CTA | Low | Add: "Learn more", "Discover", "Get started" |
| Duplicated | Medium | Write unique description per page |

### Header Issues

| Issue | Impact | Quick Fix |
|-------|--------|-----------|
| Missing H1 | Critical | Add one H1 with primary keyword |
| Multiple H1s | High | Keep one H1, convert others to H2 |
| Skipped levels | Medium | Use sequential hierarchy: H1→H2→H3 |
| Non-descriptive | Medium | Rewrite with keyword variations |
| No H2s (long block) | Medium | Add H2 every 200-300 words |

### Content Issues

| Issue | Impact | Quick Fix |
|-------|--------|-----------|
| Thin (<300 words) | Critical | Expand with subtopics, FAQ, examples |
| Keyword stuffing (>3%) | High | Reduce usage, use synonyms |
| No structured data | Medium | Add FAQ, HowTo, or Article schema |
| Missing internal links | Medium | Add 3-5 contextual links |
| No E-E-A-T signals | Medium | Add author bio, first-person narrative, citations |

### Image Issues

| Issue | Impact | Quick Fix |
|-------|--------|-----------|
| Missing alt text | Medium | Add descriptive "[subject] [context]" text |
| Generic alt ("image1") | Low | Rewrite to describe the image specifically |
| Uncompressed image (>500KB) | High | Compress to <200KB WebP |
| Generic filename (IMG_4532) | Low | Rename to descriptive-keyword-slug.webp |

```markdown
## Overall Score: 71/100

Score Breakdown:

██████░░░░ Images:           6/10  (10%)
███████░░░ Technical:        7/10  (10%)

Grade: B

## Priority Issues

### 🔴 Critical
1. **Internal linking severely underdeveloped** — Only 2 internal links found in a 2,400-word article. Add links to individual headphone review pages (/sony-wh1000xm5-review, /bose-qc-ultra-review) and the headphones category page. Target 5-8 contextual links.
2. **3 product images missing alt text** — Images for Sony WH-1000XM5, Bose QC Ultra, and Apple AirPods Max have empty alt attributes. Each missing alt is a lost signal for Google Images.

### 🟡 Important
1. **Meta description lacks CTA** — Current description states facts but doesn't compel clicks. Add "Compare prices and features" or "See our top picks."

## Quick Wins

1. **Add alt text to 3 images** (5 min): Use "Sony WH-1000XM5 noise cancelling headphones on desk" not empty attributes.
2. **Rewrite meta description** (5 min): "Compare the 10 best noise cancelling headphones for 2025. Expert-tested picks from Sony, Bose, and Apple with pros, cons, and pricing. See our top picks." (156 chars)
3. **Add 4+ internal links** (10 min): Link product names to individual review pages; add a "See all headphones" link to category hub.
```

## Page-Type Checklists

### Blog Post

```markdown
- [ ] Title includes keyword and is compelling (50-60 chars)
- [ ] Meta description has keyword and CTA (150-160 chars)
- [ ] Single H1 with keyword
- [ ] H2s cover main subtopics with keyword variations
- [ ] Keyword in first 100 words
- [ ] 1,500+ words for competitive informational queries
- [ ] 3+ internal links with descriptive anchor text
- [ ] All images have descriptive alt text
- [ ] FAQ section with FAQPage schema
- [ ] Author byline with credentials/bio
- [ ] Publication/update date visible
```

### Product Page

```markdown
- [ ] Product name in title
- [ ] Price and availability mentioned in meta description
- [ ] H1 is the product name
- [ ] Product features and benefits in H2s
- [ ] Multiple product images with descriptive alt text
- [ ] Customer reviews visible on page
- [ ] Product schema (with price, availability, rating)
- [ ] Related products internally linked
- [ ] Clear primary CTA button
```

### Landing Page

```markdown
- [ ] Keyword-optimized title with clear value proposition
- [ ] Benefit-focused meta description with CTA
- [ ] H1 communicates the primary value proposition
- [ ] Supporting H2 sections (features, social proof, FAQ)
- [ ] Trust signals (testimonials, logos, awards)
- [ ] Single primary CTA (no competing actions)
- [ ] LCP ≤2.5s (speed is critical for conversions)
- [ ] Fully mobile-responsive layout
- [ ] Above-fold content optimized for the keyword
```

## Weight Adjustments by Page Type

The default weights suit most content pages. Consider adjusting for:

| Page Type | Increase Weight | Decrease Weight | Reason |
|-----------|----------------|----------------|--------|
| E-commerce product | Image Optimization, Technical | Content Quality | Products need visuals and speed |
| Long-form guide | Content Quality, Keywords | Image Optimization | Content depth matters most |
| Landing page | Technical, Title | Content Quality | Speed and CTR drive conversions |
| Local service page | Technical, Links | Keywords | NAP consistency, local signals |

Always document weight adjustments in the audit report.
