# Writing Style Learning

Use this reference when draft quality problems are mostly about voice fit rather than message logic.

## Symptoms

Reach for a style artifact when feedback sounds like:

- "This doesn't sound like me"
- "Too polished / too AI"
- "Too salesy"
- "I would never say it like that"
- "The structure is fine, but the tone is off"

## Root-cause test

Ask:

1. Was the task logic correct?
2. Was the CTA or structure reasonable?
3. Did the draft still fail because the wording felt unlike the author?

If yes, the durable fix is usually a style guide, not more workflow bullets.

## Source order

Use these sources in order:

1. current `writing-style-guide.md`
2. recent approved or sent same-channel writing
3. recent first-party public writing
4. approved templates

Never learn from recipient-authored text, quoted replies, signatures, disclaimers, or obvious boilerplate.

## What to extract

- average length
- openings and closings
- warmth vs directness
- punctuation habits
- favored phrases
- forbidden or avoided phrases
- channel-specific differences

## Output choice

- One-off need: create a short voice brief
- Repeated drafting need: create or update `writing-style-guide.md`

Always structure the persistent guide as:

1. `## General`
2. one section per important channel

Each section should contain concrete rules the drafting skill can apply.
