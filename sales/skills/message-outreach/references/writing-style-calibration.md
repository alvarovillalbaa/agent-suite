# Writing Style Calibration

Use this reference when the user wants outreach to sound like a real sender rather than a generic sales template.

## Goal

Build the smallest durable voice artifact that improves the current draft:

- a quick voice brief for a one-off message set
- or a persistent `writing-style-guide.md` when the user will draft repeatedly

## Source priority

Use sources in this order:

1. Existing `writing-style-guide.md` or brand voice doc
2. Recent approved or sent messages from the same sender on the same channel
3. Other recent first-party writing from the sender such as LinkedIn posts, memos, or public writing
4. Existing templates already known to be approved for this sender

Do not learn voice from inbound messages, recipient text, quoted replies, signatures, legal footers, or auto-generated copy.

## Sampling rules

- Bias recent content over old content.
- Prefer 10-30 strong samples over a huge noisy corpus for a quick brief.
- For email, favor sent mail that looks human-authored and substantive.
- For LinkedIn, favor original posts and DMs, not reposts.
- For call openers, derive from email and message patterns; do not invent a stage voice the sender never uses.

## What to analyze

For each channel, capture:

- median length and paragraph shape
- openings and greetings
- closings and sign-offs
- level of warmth, directness, and formality
- punctuation habits
- repeated phrases, contractions, abbreviations, and intensifiers
- what the sender consistently avoids

Also capture a **General** layer for patterns that show up everywhere.

## Fast voice brief

When the task is a one-off or quick turnaround, return:

```markdown
## Voice brief
- General voice: [3-5 bullets]
- Email-specific rules: [bullets]
- LinkedIn-specific rules: [bullets if relevant]
- Phrases to prefer: [list]
- Phrases to avoid: [list]
- Confidence: [high / medium / low]
```

## Persistent style guide

When the user will keep drafting from the same sender, create or update `writing-style-guide.md` with:

1. `## General`
2. `## Email`
3. `## LinkedIn`
4. `## Call / spoken opener` if the sender uses a distinct spoken style

Each section should include:

- key patterns
- rules for drafting
- how this channel differs from the general voice
- short anonymized snippets only when needed to illustrate a pattern

Always use **General + closest channel section** when drafting.

## Guardrails

- Match the sender's voice; do not intensify it into caricature.
- Do not copy private or sensitive language verbatim from historical samples unless the user explicitly asks for reuse.
- If the voice evidence is thin, say the match is approximate.
- If the sender's historical writing is low quality, preserve the human feel but quietly improve clarity and correctness.
