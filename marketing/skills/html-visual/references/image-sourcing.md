# Image Sourcing

Use imagery only when it improves comprehension, framing, or credibility.

## Available Image Paths

1. Existing external image URLs already present in the repo
2. Existing local repo images
3. AI-generated images created during the task
4. Code-as-image product visuals
5. Mixed compositions

## Decision Rules

1. Prefer stable external URLs when the exact screenshot, logo, or photo already exists in the codebase.
2. Local repo images are fine for internal or quick-turn artifacts, but if the page is intended for hosted delivery, prefer moving them into the active public asset path or hosted URL.
3. Use code-as-image for product-heavy panels, dashboards, and UI explainers where precision matters.
4. Use AI-generated images for missing supporting artwork, backgrounds, or atmosphere.
5. Combine a generated background with a coded product foreground when the page needs both mood and product fidelity.
